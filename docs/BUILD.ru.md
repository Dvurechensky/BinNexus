## Важно

BinNexus использует `dumpbin`, поэтому запуск должен выполняться из (на выбор):

- x86 Native Tools Command Prompt
- x64 Native Tools Command Prompt
- Developer Command Prompt for Visual Studio

---

## Установка

```sh
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
```

## Запуск

### Правила использования

| Аргумент                       | Описание                                                                  |
| ------------------------------ | ------------------------------------------------------------------------- |
| `-h, --help`                   | Показать справку и выйти                                                  |
| `--input PATH`                 | Путь к папке с бинарными файлами (DLL/EXE) для анализа                    |
| `--output DIR`                 | Директория для генерации портала (по умолчанию: `build`)                  |
| `--exclude-system`             | Исключить системные DLL Windows (kernel32, user32 и др.)                  |
| `--exclude-crt`                | Исключить библиотеки рантайма C/C++ (msvcrt, msvcp, api-ms-win-crt-\*)    |
| `--lang {en,ru}`               | Язык портала (по умолчанию: `en`)                                         |
| `--skip-dirs DIR...`           | Директории, пропускаемые при сканировании (по умолчанию: `legacy`)        |
| `--build-portal`               | Построить HTML-портал из существующего `graph.json` в выходной директории |
| `--version`                    | Показать версию программы и выйти                                         |
| `--runtime`                    | Включить анализ модулей в рантайме через отладчик (x32dbg/x64dbg)         |
| `--runtime-include-system`     | Включить системные DLL в результаты рантайма (требует `--runtime`)        |
| `--runtime-timeout MS`         | Таймаут анализа рантайма на один бинарник (по умолчанию: `3000` мс)       |
| `--runtime-limit N`            | Максимальное число бинарников для анализа в рантайме (по умолчанию: `50`) |
| `--runtime-mode {all,dll,exe}` | Какие бинарники анализировать в рантайме (по умолчанию: `all`)            |
| `--runtime-dump`               | Сохранять сырые списки модулей рантайма для каждого бинарника             |
| `--runtime-verbose`            | Включить подробное логирование анализа в рантайме                         |

### Пример старта

> в корневом репозитории

```sh
python -m app.binnexus.cli.main --input "F:\LIZERIUM\LizeriumFreelancerMode" --output "build" --exclude-system --exclude-crt --lang en --skip-dirs plugins --runtime --runtime-limit 100 --runtime-mode all --runtime-timeout 3000 --runtime-verbose --runtime-dump
```

## Создание портала пользовательского интерфейса

> в корневом репозитории

```sh
python -m app.binnexus.cli.main --build-portal
```

---

## Запуск портала

```sh
cd build/portal
python -m http.server 16801
```

Открыть:

```
http://localhost:16801
```

---

## Проверка dumpbin

```sh
dumpbin
```

Если команда не найдена — значит запущена неправильная консоль.

---

## Примечания

> [!TIP]
> Используйте x86 toolchain для старых игр и legacy DLL.

> [!WARNING]
> Не запускайте анализ system32 — это создаст огромное количество шума.

> [!NOTE]
> Рекомендуется исключать директории plugins / legacy для чистоты графа.
